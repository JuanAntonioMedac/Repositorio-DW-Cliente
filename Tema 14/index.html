Tarea: Almacenamiento Web

Prácticas con localStorage y sessionStorage

Ejercicio 1 – Preferencias de usuario con localStorage

Objetivo: Trabajar el almacenamiento persistente de datos en el navegador.

Enunciado: Crea una página web que permita al usuario introducir su nombre y elegir un color de fondo para la página. Ambos datos deben guardarse utilizando localStorage.

Requisitos:

  • Al recargar la página, los datos deben mantenerse.
  • El color de fondo debe aplicarse automáticamente al cargar la web.
  • Incluir un botón “Restablecer preferencias” que elimine los datos almacenados.

Ejercicio 2 – Contador de visitas con localStorage

Objetivo: Reforzar la lectura, escritura y actualización de datos almacenados.

Enunciado: Implementa un contador que muestre cuántas veces se ha visitado la página desde ese navegador.

Requisitos:

  • El contador debe incrementarse automáticamente cada vez que se recarga la página.
  • Mostrar el número de visitas en pantalla.
  • Incluir un botón para reiniciar el contador.

Visitas: 0

Ejercicio 3 – Almacenamiento temporal con sessionStorage

Objetivo: Comprender el funcionamiento del almacenamiento temporal.

Enunciado: Crea un formulario con los campos: Nombre y Curso. Los datos deben guardarse usando sessionStorage.

Requisitos:

  • Al recargar la página, los datos deben seguir visibles.
  • Al cerrar la pestaña o el navegador, los datos deben desaparecer.
  • Añadir un texto explicativo indicando por qué ocurre esto.

Nota: Los datos de sessionStorage se mantienen al recargar, pero se pierden al cerrar la pestaña porque su ciclo de vida termina con la sesión del navegador.

Ejercicio 4 – Comparativa localStorage vs sessionStorage

Objetivo: Comparar de forma práctica ambos sistemas de almacenamiento.

Enunciado: Crea una página con un campo de texto y dos botones: Guardar en localStorage y Guardar en sessionStorage. El mismo dato se guardará en ambos sistemas.

Requisitos:

  • Mostrar en pantalla el valor almacenado en cada uno.
  • Comprobar y explicar qué ocurre al recargar la página, cerrar la pestaña y abrir una nueva pestaña.

Nota: localStorage persiste aunque se cierre el navegador, mientras que sessionStorage se pierde al cerrar la pestaña.

Ejercicio 5 – Lista de tareas con localStorage

Objetivo: Trabajar con arrays y objetos almacenados en el navegador.

Enunciado: Desarrolla una pequeña aplicación de lista de tareas que permita añadir, mostrar y eliminar tareas. Las tareas deben almacenarse en localStorage.

Requisitos:

  • Usar un array de objetos.
  • Convertir los datos con JSON.stringify() y JSON.parse().
  • Mantener la lista tras recargar la página.
  • Si hay varias pestañas abiertas, deben sincronizarse automáticamente.
